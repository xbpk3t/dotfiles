  也就是说：当前 nixpkgs 更新后默认 DBus 实现从 dbus 变成了 dbus-broker，NixOS 认为这是 critical
  component change，不允许普通 switch 直接切过去。deploy-rs --skip-checks 只跳过 deploy-rs 自己的
  checks，不会跳过 NixOS activation 的 switchInhibitors。

  后面的这些：

  Attempting to re-activate the last generation

  更像是 deploy-rs rollback/activation waiter 的次生失败。当前我查了两台远端 systemctl --failed，没
  有残留 failed units，所以机器现在大概率已经回到旧 generation，服务状态没有明显挂住。

  你暂时不需要补充太多 context。若要继续精确处理，我只需要你确认想走哪条路：

  1. 推荐：按 NixOS 提示走 boot + reboot，让 dbus -> broker 在重启边界完成。
  2. 保守：先在 VPS 配置里显式 pin services.dbus.implementation = "dbus"，恢复 deploy-rs switch；之
     后单独安排一次 boot/reboot 迁移到 broker。
  3. 强行 switch：给 activation 带 NIXOS_NO_CHECK=1，但这正是日志警告的风险路径，我不建议在远端 VPS
